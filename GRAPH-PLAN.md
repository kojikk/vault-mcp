# GRAPH-PLAN.md — граф-слой для vault-mcp

> План разработки. Цель: превратить вольт из набора заметок в **запрашиваемый граф знаний для агента** — без UI, без внешних LLM-пайплайнов, без нарушения security-модели. Принципы токенизации и связи концептов взяты из graphify (github.com/safishamsi/graphify, MIT), реализация — нативная в TypeScript.

Статус: фазы 0–4 реализованы и задеплоены 2026-06-11 (v0.2.0, 27 тулз, 80 тестов). Осталось: правки конфига second-brain-bot (отложены вместе с переработкой бота) и наблюдение за качеством IDF-match на русской морфологии (§13.1).

---

## 0. Что берём из graphify и что нет

**Берём (принципы, портируем в TS):**
- IDF-взвешенное сопоставление NL-запроса с узлами графа (токенизация, нормализация диакритики/регистра).
- BFS/DFS-обход подграфа с `token_budget` и демпфированием хаб-узлов (cap по p99 распределения степеней).
- Confidence-теги на рёбрах: `extracted` / `inferred` / `ambiguous`.
- Community detection (у них Leiden; у нас Louvain через graphology — на личном масштабе разница косметическая).
- Санитизация LLM-derived строк перед рендером (их урок F-010 — инъекции через label'ы).
- Идея `GRAPH_REPORT`-аналитики: god nodes, неожиданные связи → сырьё для lint.

**Не берём:**
- Python-рантайм и их MCP-сервер (сайдкар отменён: лишний процесс + LLM-egress для markdown).
- LLM-экстракцию через внешний API: **наш LLM — это сам агент**, который и так читает заметку в момент ingest. Экстракция связей — шаг ingest-флоу, маржинальная стоимость ~0.
- tree-sitter / графы кода. Для кодовых проектов graphify остаётся **CLI по требованию** (код парсится локально без LLM): прогон по проекту → `GRAPH_REPORT.md` → обычный ingest отчёта в вольт.
- Визуализацию (graph.html). Obsidian graph view покрывает UI-потребность сам.

## 1. Инварианты реализации

1. **Security-модель не трогаем:** no egress, единственный child process — ripgrep, git — isomorphic-git. Граф-слой — чистый TS + чтение/append файлов внутри VAULT_ROOT.
2. **Два слоя графа:**
   - **derived** — детерминированно собирается из файлов (wikilinks, embeds, теги, frontmatter). Не хранится как истина, не может разойтись с вольтом, покрывает 100% заметок с нулевой стоимостью.
   - **semantic** — типизированные рёбра, которые агент пишет при ingest (`graph_upsert`). Хранится, версионируется git'ом, валидируется lint'ом.
3. **Граф — индекс, не истина.** Первоисточник — заметки. Агент относится к подграфу как к hot-кэшу: ориентир, проверяемый чтением страниц.
4. **Личный масштаб:** тысячи узлов, десятки тысяч рёбер. Всё в памяти, хранение — JSONL. Никаких Neo4j/SQLite в v1.
5. Содержимое графа (label'ы пришли из контента вольта) — **untrusted data**: рендер обёрнут в ту же UNTRUSTED-обёртку, что и read_file.

## 2. Хранение

```
_system/graph/semantic.jsonl    # append-only типизированные рёбра (пишет graph_upsert)
```

Строка semantic.jsonl (zod-схема):

```json
{"src":"Знания/AI/RAG.md","tgt":"Знания/AI/Эмбеддинги.md","relation":"использует","confidence":"extracted","source_note":"_raw/articles/2026-06-01-rag.md","created":"2026-06-11","key":"<idempotency>"}
```

- `src`/`tgt` — vault-relative пути заметок **или** голые label'ы концептов без страницы (label без страницы = сигнал «кандидат на entity page», его ловит lint).
- v1 append-only: рёбра не удаляются и не редактируются; ошибочное ребро гасится записью с `relation:"retracted"` (assemble-слой фильтрует). Пересмотр — открытый вопрос (§13).

**Derived-слой на диск не пишется.** Собирается при старте сервера полным обходом `.md` (вне reserved-зон + `Знания/` и организационные узлы; `_raw/` и `_attachments/` в граф не входят — это сырьё), инкрементально обновляется в write-path инструментов. Холодная сборка на тысячах файлов — секунды, кэш не нужен.

## 3. Новые модули — `src/core/graph/`

| Модуль | Ответственность |
|---|---|
| `linkparse.ts` | Единый парсер `[[wikilink]]` / `![[embed]]` / `[md](path.md)` / `#тегов` / frontmatter. **Выносится из backlinks.ts** (там уже есть regex'ы — рефакторинг, не дублирование). |
| `builder.ts` | Derived-слой: обход вольта → узлы (заметки, теги, entity по frontmatter `type/domain`) + рёбра (link/embed/tag-membership). Инкрементальный апдейт по одному файлу. |
| `store.ts` | semantic.jsonl: чтение при старте, append через atomic.ts + lock.ts, zod-валидация, лимит размера. |
| `assemble.ts` | Слияние derived + semantic в рабочий граф (adjacency Map), фильтр `retracted`, степени узлов, p99-порог хабов. |
| `match.ts` | Нормализация label (lowercase, NFD-диакритика, ё→е), токенизация, IDF-веса по корпусу label'ов + алиасов (frontmatter `aliases`), скоринг запроса. |
| `traverse.ts` | BFS/DFS от matched-узлов: `depth`, `token_budget`, демпфирование хабов, фильтр по relation. |
| `communities.ts` | Louvain (`graphology` + `graphology-communities-louvain`), god nodes, межсообщественные «мосты». Фаза 3. |
| `render.ts` | Текстовый рендер подграфа/пути/соседей с confidence-тегами; санитизация label'ов; UNTRUSTED-обёртка. |

Новые зависимости: `graphology`, `graphology-communities-louvain` (фаза 3), `pdfjs-dist` (фаза 4). Все — pure JS, без нативных бинарей и postinstall-скриптов (проверить при добавлении).

## 4. Новые инструменты — `src/tools/graph.ts`

**Read-only** (без подтверждений, доступны боту автоматически):

| Тулза | Аргументы | Возвращает |
|---|---|---|
| `graph_query` | `question`, `depth=2`, `token_budget=2000` | Подграф вокруг matched-узлов: узлы + рёбра с relation/confidence, ссылки `[[...]]` на страницы. Описание тулзы — триггерное: «Use FIRST when answering any question about the user's knowledge, projects or how concepts relate». |
| `graph_neighbors` | `node`, `relation?` | Соседи узла с типами связей и источником ребра (derived/semantic). |
| `graph_path` | `source`, `target`, `max_hops=6` | Кратчайший путь между концептами или честное «не связаны». |
| `graph_stats` | — | Узлы/рёбра по слоям, покрытие entity-страниц семантическими рёбрами, top-связные узлы, число label'ов без страниц. |

**Additive** (через IdempotencyLedger, как add_raw):

| Тулза | Аргументы | Поведение |
|---|---|---|
| `graph_upsert` | `edges[]`, `idempotency_key` | Валидация схемы → append в semantic.jsonl → git-коммит (существующий путь мутаций). Не структурная: рёбра аддитивны и «обратимы» через retracted, dry-run/confirm не нужен. |

Регистрация: `registerGraphTools(server, ctx, ledger)` в [mcp.ts](src/mcp.ts).

## 5. Правки существующего кода

1. **`mcp.ts` — поле `instructions`** в `McpServer` options (сейчас не используется!). Это системный промпт, который MCP-клиенты (Claude Code, Claude Desktop, Claude chat-connectors) получают при initialize — главный рычаг enforcement (§8). Содержимое: сжатая лестница чтения + «вопросы о проектах/знаниях пользователя → сначала `read_hot` + `graph_query`» + граница доверия.
2. **`core/lint.ts` — граф-секция отчёта:** рёбра semantic с несуществующими концами; entity-страницы с 0 семантических рёбер (непокрытое знание); label'ы-концепты без страниц (кандидаты на create_note); счётчик покрытия. Плюс перенос идеи «surprising connections»: пары сообществ с единственным ребром-мостом.
3. **`tools/read.ts` — `read_hot`:** в конец ответа дописывается graph digest (3–5 строк: размер графа, свежие semantic-рёбра, топ-сообщества). Агент получает граф в лицо на первом же чтении, не меняя поведения. Фаза 3, аккуратно с токенами (~150 токенов максимум).
4. **`tools/write.ts` / `edit.ts` / `structural.ts`:** после мутации `.md` — инкрементальный апдейт derived-слоя (вызов builder на затронутых путях). move/promote дополнительно ремапят пути в in-memory графе (semantic.jsonl при переносе страниц чинится так же, как backlinks — добавить в план move).
5. **Версия** `vault-mcp` → 0.2.0.

## 6. PDF-экстракция (фаза 4)

Тулза `extract_pdf(path)` — `path` внутри `_attachments/`, `pdfjs-dist` (pure JS) → текст → ответ агенту **плюс** опция `save_raw:true` → сразу `add_raw` в `_raw/docs/` со ссылкой на оригинал. Дальше — обычный ingest по agent.md §7.

Ограничения честно: сканы без текстового слоя → ошибка «текстовый слой отсутствует, нужен OCR — вне скоупа»; лимит размера файла; результат — untrusted data.

DOCX/XLSX — не в этой фазе (отдельные либы, отдельное решение). Картинки — мультимодальность агента, MCP не конвертирует (как сейчас в agent.md §7.2).

## 8. Enforcement — как заставить Claude Code и Claude-чат ходить в граф

Диагноз честный: сейчас единственный механизм — auto-memory (рекомендация, не контракт) + скилл-триггеры. В сессии 2026-06-11 агент (я) работал из памяти и читал репо напрямую, vault-тулзы не звал. Для repo-задач это норм, для вопросов о знаниях/проектах — дыра.

Слои, от детерминированного к мягкому:

1. **Дизайн тулзов (самый сильный — не требует послушания агента):**
   - описания тулз = триггеры («Read this first» у read_hot уже работает — расширить паттерн на graph_query);
   - graph digest внутри `read_hot` (§5.3): агент, который зовёт хотя бы read_hot, видит граф автоматически;
   - (фаза 3+, по результатам) аннотация хитов `search` графовыми соседями.
2. **MCP server instructions (§5.1):** инструкция приезжает в систем-промпт **любого** MCP-клиента — Claude Code, Claude Desktop, Claude-чат с коннектором — без правок на стороне клиентов. Сейчас поле пустует; это главный недоиспользованный рычаг.
3. **Claude Code:** секция в глобальном `~/.claude/CLAUDE.md` — «вопросы о проектах/жизни/знаниях Кирилла → сначала вольт (read_hot + graph_query), потом общие знания; repo-задачи — код напрямую». CLAUDE.md инжектится как override-инструкция и весит больше auto-memory.
4. **Скилл `anthropic-skills:obsidian`:** обновить тело — маршрут Query через graph_query, маршрут ingest с graph_upsert.
5. **Бот (second-brain-bot):** систем-промпт = живой agent.md → подтянет v2 сам. Правки конфига: `WRITE_IDEM_TOOLS` += `graph_upsert` (+ давно отложенные `mark_raw_ingested`, `append_contradiction`); `STRUCTURAL_TOOLS` += `edit_note`.
6. **Auto-memory:** обновить `reference_vault_knowledge_base.md` новой лестницей.

Ограничение: ни один слой не даёт 100% — поэтому ставка на слой 1 (граф встроен в тулзы, которые агент и так зовёт), остальные — страховка.

**Топология (решение Кирилла, 2026-06-11): LAN — это фича, не баг.** vault-mcp слушает только LAN (`192.168.1.50:8788`); доступ с телефона идёт через second-brain-bot, который находится в одной сети с MCP. Публичный endpoint наружу не выставляем. Переработка бота (в т.ч. подхват graph-тулз в конфиге) — отдельная задача позже.

## 9. Bootstrap существующего вольта

1. **Derived:** автоматически при первом старте 0.2.0 — мгновенно, весь вольт.
2. **Semantic:** разовая сессия Claude Code. Дёшево, потому что entity-страницы **уже содержат** полуструктурированные связи — секция «Связанные концепты» формата `[[X]] — как связан`. Проход: `_index` → entity-страницы → конвертация секций в `graph_upsert` (relation из текста «как связан», confidence=extracted). Новый LLM-анализ нужен только там, где секция пустая.
3. Остальной вольт — лениво: связи дописываются при каждом касании заметки в обычных флоу.

## 11. Тесты и security-чеклист

vitest, по образцу существующих:
- `linkparse`: wikilinks с алиасами/заголовками, embeds, md-links, теги, frontmatter; злые имена файлов.
- `builder`: derived-граф на фикстурном вольте; инкрементальный апдейт = полная пересборка (эквивалентность).
- `match`: IDF-ранжирование, диакритика, ё/е, алиасы; пустой результат.
- `traverse`: depth, token_budget обрезает честно, хабы демпфируются, relation-фильтр.
- `store`: append + idempotency-повтор = одна запись; битая строка jsonl не валит загрузку (skip + warn); retracted фильтруется.
- `tools`: контракты, UNTRUSTED-обёртка, label с `[[инъекцией]]`/markdown в рендере экранирован.

Security-чеклист перед мержем каждой фазы: нет новых child processes; нет network-вызовов; новые deps — pure JS без postinstall; все пути через paths.ts; semantic.jsonl ограничен по размеру; вывод графа — untrusted-обёртка.

## 12. Вне скоупа (зафиксировано)

- Графы кода/SQL — graphify CLI по требованию на проектах, отчёт → ingest.
- OCR, видео, docx/xlsx-экстракция.
- Визуализация (Obsidian graph view достаточно; canvas-экспорт сообществ — возможная фаза 5).
- Эмбеддинги/векторный поиск — отдельное решение, если IDF-match окажется слаб на русской морфологии.
- Публичный endpoint для claude.ai-чата (§8, решение пользователя).

## 13. Открытые вопросы

1. **Русская морфология в match:** v1 — нормализация + алиасы во frontmatter; если мало — лёгкий стеммер (snowball-ru, pure JS) фазой 3.5.
2. **Ретракция рёбер:** хватает ли `relation:"retracted"` или нужен полноценный supersede (новое ребро заменяет старое по key)?
3. **Двунаправленность:** derived-рёбра направленные (A ссылается на B); semantic — хранить направленно, traverse по умолчанию обходит как ненаправленный (как graphify `_src`/`_tgt`)?
4. **Digest в read_hot:** не раздует ли токены лестницы — мерить после фазы 3, выкинуть при сомнении.
